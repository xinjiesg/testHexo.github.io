4. Queue

发表于 2018-01-27

1. What is queue

  • A queue is a list where can only insert new items at the back and remove items from the front;
  • This ensures that First come, first serve!

2. Why use queue

  • When you want to add objects to a temporary list, and then pull them off later;
  • Often, the order in which you add and remove these objects matter.
  • FIFO or first-in, first-out order
  • (Same reason as stack, just order different)

3. Implementation

3.1 Enqueue

  • You’ll use the LinkedList to store items in queue.
  • This method will mutate the LinkedList so need add the mutating keyword;

3.2 Dequeue

  • Returns the first item in the queue, the return type is nullable to handle the queue being empty.
  • need add the mutating keyword;
  • Using the guard statement handle the queue being empty

3.3 Peek

  • Returns the item at the beginning of the queue without removing it.
  • No mutating
  • No guard statement, as no need to remove item;
