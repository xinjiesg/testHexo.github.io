3. Stack

发表于 2018-01-25

1. What is stack

  • Stacks are like arrays, but with limited functionality;
  • Push to add a element to the top of the stack.
  • Pop to remove the element from the top;
  • Peek at the top element without popping it off.

2. Why use stack

  • When you want to add objects to a temporary list, and then pull them off later;
  • Often, the order in which you add and remove these objects matter.
  • LIFO or last-in first-out order

3. Implementation

3.1 Declare

Declare with an array

3.2 Push

  • a push operation puts the new element at the end of the array, not the beginning. Inserting at the beginning of an array is expensive, an O(n) operation, because it requires all existing array elements to be shifted in memory. Adding at the end is O(1);

3.3 Pop

  • The pop method returns an optional String. If you try to pop an empty stack, then you’ll get a nil.
  • The Swift array has a handy method for removing it’s last element: popLast.

3.4 Peak

  • Peek avoids mutating the contents, hence the mutating keyword isn’t necessary in this case.
